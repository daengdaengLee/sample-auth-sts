# AGENTS.md

이 저장소에서 작업할 때 따르는 규칙.

## 문서 표기 규칙

### 키보드로 바로 입력할 수 없는 기호를 쓰지 않는다

문서(`README.md` 등)에는 키보드로 바로 입력할 수 없는 특수 기호를 쓰지 않는다.
일반 ASCII 문자로, 그리고 **기존 문서가 이미 쓰는 표기를 우선** 따라 대체한다.

자주 나오는 기호 -> 대체 (전부는 아님):

- 가운뎃점 `·` (U+00B7) -> 등위 접속은 `/` (예: `양도/재사용`. 기존 `동적/단기`, `서버/클라이언트`)
- 엠대시 `—` (U+2014) -> 삽입구는 괄호 `( )`
  (예: `완화책(...)을`. 기존 `증명(Proof of Possession)하여`처럼 닫는 괄호 뒤 조사/어미를 바로 붙임)
- 굽은 따옴표 `“ ” ‘ ’` -> ASCII `"` `'`
- 엔대시 `–` (U+2013) -> 하이픈 `-`
- 말줄임표 `…` (U+2026) -> 마침표 세 개 `...`
- 화살표 `→` `⇒` 등 -> `->`

작성 후 비키보드 기호가 섞이지 않았는지 확인한다
(ASCII + 한글 외 문자를 검출하는 코드포인트 스캔 등).

(이 `AGENTS.md`의 백틱 예시에 든 기호 자체는 규칙을 설명하기 위한 것이므로 예외다. 고치지 않는다.)

### 마크다운 표는 코드포인트(문자 수) 기준으로 정렬한다

이 저장소의 표는 디스플레이 폭이 아니라 **코드포인트(문자 수)** 기준으로 정렬한다
(JetBrains 마크다운 포매터 관례). 한글/전각 문자도 폭 2가 아니라 **1 문자**로 센다.

- 각 셀은 공백으로 채워 열 너비를 문자 수 기준으로 맞춘다
  (`| ` + 내용 + 공백 패딩 + ` |`).
- 구분자 행의 대시 개수 = 해당 열의 최대 문자 수 + 2.
- 결과적으로 표의 모든 행(소스 줄)이 같은 문자 길이를 갖는다.
- 한글이 섞이면 눈으로는 파이프가 어긋나 보이지만(한글이 2칸 차지) 정상이다.
  문자 수만 맞으면 된다.

## 코드 규칙

### 로깅: 요청 흐름 로거에 WithGroup 을 쓰지 않는다

요청 흐름 로거(`logging.New` 또는 `slog.Default`, 즉 `ContextHandler` 로 감싼 로거)에는
`WithGroup` 을 쓰지 않는다. context 상관관계 속성(request_id 등)은 로그 시점에 레코드로
추가되므로, 그룹이 열린 상태에서 로깅하면 그 속성이 그룹 안으로 중첩된다(예:
`g.request_id`). 필드 묶음이 필요하면 호출 시점에 inline `slog.Group(...)` 을 쓴다.
`With(...)` 로 속성을 더하는 것은 최상위로 남으므로 무방하다.
